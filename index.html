System Architecture

Fully Decoupled Microservices

Each layer runs as an independent unit—scale the API, dashboard, or database without touching anything else.

🧩

Chrome Extension

Records raw mouse positions, scroll offsets, and mock eye-gaze using DOM listeners. Batches events every 5 seconds via the background service worker.

⚡

NestJS Backend API

Exposes /auth/login and /track/events REST endpoints backed by Prisma and PostgreSQL.

PORT 3001
📊

Next.js Dashboard

App Router application with cookie-based auth, session list view, and a canvas-based heatmap replayer powered by simpleheat.

PORT 3000
🗄️

PostgreSQL Database

A 3-table relational schema storing Users, Sessions, and EventLogs. Accessed via Prisma ORM with the pg native pool adapter.

Platform Features

Everything You Need

Purpose-built to handle thousands of concurrent tracking events without compromising performance.

🔐

Direct Injection Auth

No JWT overhead. The extension stores user.id from a single POST to the backend — efficient and stateless.

📦

5-Second Batched Sync

Raw events are buffered in memory and flushed to the API every 5,000ms to prevent server congestion from high-frequency mouse ticks.

🍪

HTTP-Only Cookie Auth

The Next.js proxy stamps a secure dashboard_user cookie to protect server-rendered dashboard routes.

🎨

Canvas Heatmap Replay

Uses simpleheat to visualize event data on an HTML5 canvas, matching original timestamp intervals for authentic replay.

👁️

Eye-Gaze Tracking

Hooks into WebGazer / FaceMesh overlays to capture where users are looking, not just where they click.

🚀

Vercel + Render Deploy

Ship the Next.js dashboard on Vercel and the NestJS API on Render. Both zero-config cloud platforms.

Getting Started

Up and Running in Minutes

Requires PostgreSQL, Node.js 20+, and a Chromium-based browser.

1

Clone the Repository

Clone the project and navigate into the root directory.

git clone https://github.com/your-username/User_attention_heatmap2.git && cd User_attention_heatmap2
2

Configure Environment Variables

Set your PostgreSQL connection string in both backend/.env and dashboard/.env.

DATABASE_URL="postgresql://user:password@localhost:5432/heatmap"
3

Initialize the Database

Run Prisma migrations from inside the backend folder to create all tables.

cd backend && npx prisma migrate dev --name init
4

Start the Backend API

Boot the NestJS server. It listens on port 3001 with CORS open to all origins.

npm run start:dev
5

Start the Dashboard

Open a new terminal, navigate to the dashboard folder, and run the dev server on port 3000.

cd ../dashboard && npm run dev
6

Load the Chrome Extension

Open chrome://extensions, enable Developer Mode, click "Load unpacked" and select the project root folder.
