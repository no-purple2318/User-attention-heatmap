Platform Features

Built for Performance & Precision

From batch ingestion to live canvas replay — every component is designed to be efficient and easy to extend.

⚡

5-Second Batched Sync

Events are buffered in background.js and flushed as a single POST every 5,000ms, protecting the API from per-tick load.

🔐

Email-Based Auth

The extension sends the user email to the backend once. The returned user.id is cached in chrome.storage.local, no JWT required.

🍪

HTTP-Only Cookie Session

The Next.js proxy route mints a secure dashboard_user cookie so server components can access user data safely.

🎨

simpleheat Canvas Renderer

Heatmap data is rendered using simpleheat on an HTML5 canvas with timestamp-accurate interval-based event replay.

👁️

Eye-Gaze Tracking

Integrates with WebGazer and optional face-mesh calibration to go beyond cursor data and capture actual gaze point estimation.

🗄️

PostgreSQL via Prisma

A clean 3-table schema — User, Session, EventLog — managed by Prisma ORM for type-safe queries and effortless migrations.

Getting Started

Run It Locally in 6 Steps

Requires Node.js 20+, a PostgreSQL instance, and Google Chrome.

1

Clone the Repo

Download the project and open the root directory.

git clone https://github.com/your-username/User_attention_heatmap.git && cd User_attention_heatmap
2

Configure the Database URL

Add your PostgreSQL connection string to backend/.env and dashboard/.env.

DATABASE_URL="postgresql://user:password@localhost:5432/heatmap"
3

Apply Prisma Migrations

Create all three database tables from the Prisma schema.

cd backend && npx prisma migrate dev --name init
4

Start the NestJS Backend

Boot the API on port 3001. CORS is open to all origins by default.

npm run start:dev
5

Start the Next.js Dashboard

Navigate to the dashboard directory and start the dev server on port 3000.

cd ../dashboard && npm run dev
6

Load the Chrome Extension

Open chrome://extensions → enable Developer Mode → click "Load unpacked" → select the project root folder. Enter your email in the extension popup to start tracking.

Deployment

Ship to Production

Each piece is independently deployable across best-in-class free-tier cloud providers.

🗄️ Database — Neon / Supabase

Spin up a free-tier managed PostgreSQL instance and copy the connection string into your environment variables. Run npx prisma db push to sync the schema.

Connect your GitHub repo to Render, set root directory to backend, and configure the build and start commands. Render auto-deploys on push.

render.com →

Import the same repo on Vercel with root directory set to dashboard. Add environment variables and deploy — zero-config Next.js support out of the box.

vercel.com →

In popup.js and background.js, replace localhost:3001 with your Render backend URL and reload the extension in Chrome.
